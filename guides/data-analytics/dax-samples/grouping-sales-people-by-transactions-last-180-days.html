Grouping of Sales People, by No. of Transactions made in the last 180 days

Grouping of Sales People, by No. of Transactions made in the last 180 days

This sample shows, for each Sales Person:

(i) The No. of Transactions made in the last 180 days
(ii) Grouping each Sales Person, into Groups (e.g. Top Sales People, Ok Sales People, Poor Sales People - according to the Total No. of Transactions made in the last 180 days)

Data Model - Snippet

Measures - Sales & Sales People

Last Purchase Date (from selected dates)

DAX 
Last Purchase Date (from selected dates) =
MAXX ( ALLSELECTED ( Sales_Table ), Sales_Table[Date] )

Sales People Groupings, based on the No. of Transactions Made in the Last 180 Days

DAX 
Sales People Groupings, based on the No. of Transactions Made in the Last 180 Days =
CALCULATE (
    SELECTEDVALUE ( 'Salespeople Transaction Group'[Group], BLANK () ),
    FILTER (
        ALL ( 'Salespeople Transaction Group' ),
        [Total No. of Transactions  made in the last 180 days (from selected dates)] >= 'Salespeople Transaction Group'[Min]
            && [Total No. of Transactions  made in the last 180 days (from selected dates)] < 'Salespeople Transaction Group'[Max]
    )
)

Total No. of Transactions made in the last 180 days (from selected dates)

DAX 
Total No. of Transactions  made in the last 180 days (from selected dates) =
CALCULATE (
    COUNTROWS ( Sales_Table ),
    FILTER (
        ALLSELECTED ( Calendar_Table ),
        Calendar_Table[Date] > [Last Purchase Date (from selected dates)] - 180
            && Calendar_Table[Date] <= [Last Purchase Date (from selected dates)]
    )
)

Tables

Calendar_Table

DAX 
Calendar_Table =
GENERATE (
    CALENDAR ( DATE ( 2017, 1, 1 ), DATE ( 2019, 12, 31 ) ),
    VAR currentDay = [Date]
    VAR Day =
        DAY ( currentDay )
    VAR Month =
        MONTH ( currentDay )
    VAR Year =
        YEAR ( currentDay )
    RETURN
        ROW ( "Day", day, "Month", month, "Year", year )
)
