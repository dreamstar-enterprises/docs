    Strings

    Guides • Erlang & Elixir • 04/12/2021

    Introduction

    Elixir doesn’t have a dedicated string type. Instead, strings are represented by using either a binary or a list type.

    Binary Strings

    The most common way to use strings is to specify them with the familiar double-quotes syntax:

    Elixir 
    iex(1)> "This is a string"
"This is a string"

    The result is printed as a string, but underneath it’s a binary — nothing more than a consecutive sequence of bytes.

    Strings don’t have to finish on the same line:

    Elixir 
    iex(4)> "
        This is
        a multiline string
        "

    Embedded String Expressions

    Elixir provides support for embedded string expressions. You can use #{} to place an Elixir expression in a string constant. The expression is immediately evaluated, and its string representation is placed at the corresponding location in the string:

    Elixir 
    iex(2)> "Embedded expression: #{3 + 0.14}"
"Embedded expression: 3.14"

    Escape Characters

    Classical \ escaping works

    Elixir 
    iex(3)> "\r \n \" \\"

    Sigils

    Elixir provides another syntax for declaring strings, so-called sigils. In this approach, you enclose the string inside ~s():

    Elixir 
    iex(5)> ~s(This is also a string)
"This is also a string"

    Sigils can be useful if you want to include quotes in a string:

    Elixir 
    iex(6)> ~s("Do... or do not. There is no try." -Master Yoda)
"\"Do... or do not. There is no try.\" -Master Yoda"

    There’s also an uppercase version ~S that doesn’t handle interpolation or escape characters (\):

    Elixir 
    iex(7)> ~S(Not interpolated #{3 + 0.14})
"Not interpolated \#{3 + 0.14}"

iex(8)> ~S(Not escaped \n)
"Not escaped \\n"

    Heredocs

    There’s a special heredocs syntax, which supports better formatting for multiline strings. Heredocs strings start with a triple double-quote. The ending triple double-quote must be on its own line:

    Elixir 
    iex(9)> """
        Heredoc must end on its own line """
        """
"Heredoc must end on its own line \"\"\"\n"

    Concatenation

    Because strings are binaries, you can concatenate them with the <> operator:

    Elixir 
    iex(10)> "String" <> " " <> "concatenation"
"String concatenation"

    More Information

    Helper functions are available for working with binary strings. More can be found in the String module:

    » HexDocs - String.

    Character Lists

    The alternative way of representing strings is to use single-quote syntax:

    Elixir 
    iex(1)> 'ABC'
'ABC'

    This creates a character list - a list of integers in which each element represents a single character.

    The previous result is exactly the same as if you manually construct the list of integers:

    Elixir 
    iex(2)> [65, 66, 67]
'ABC'

    The runtime doesn’t distinguish between a list of integers and a character list. When a list consists of integers that represent printable characters, it’s printed to the screen in the string form.

    Other Syntax

    Like with binary strings, there are syntax counterparts for various definitions of character lists:

    Elixir 
    iex(3)> 'Interpolation: #{3 + 0.14}'
'Interpolation: 3.14'

iex(4)> ~c(Character list sigil)
'Character list sigil'

iex(5)> ~C(Unescaped sigil #{3 + 0.14})
'Unescaped sigil \#{3 + 0.14}'

iex(6)> '''
        Heredoc
        '''
'Heredoc\n'

    Character lists aren’t compatible with binary strings. Most of the operations from the String module won’t work with character lists.

    Binary String < > Character List

    In general, you should prefer binary strings over character lists. Occasionally, some functions may work only with character lists. This mostly happens with pure Erlang libraries.

    In such cases, you can convert a binary string to a character list version using the String.to_charlist/1 function:

    Elixir 
    iex(7)> String.to_charlist("ABC")
'ABC'

    To convert a character list to a binary string, you can use List.to_string/1.

    In general, you should prefer binary strings as much as possible, using character lists only when some third-party library (most often written in pure Erlang) requires it.
