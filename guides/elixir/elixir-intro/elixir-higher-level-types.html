    Higher Level Types

    Guides Erlang & Elixir 04/12/2021

    Introduction

    Higher-level types are also available, which build on these basic types to provide additional functionality - they are higher-level abstractions of the basic types.

    Ones often used are Range, Keyword, MapSet, Date, Time, NaiveDateTime, and DateTime.

    Range

    A range is an abstraction that allows you to represent a range of numbers.

    Elixir 
    iex(1)> range = 1..2

    You can ask whether a number falls in the range by using the in operator:

    Elixir 
    iex(2)> 2 in range
true

iex(3)> -1 in range
false

    Ranges are enumerable, so functions from the Enum module know how to work with them.

    The following example uses this function with a range to print the first three natural numbers:

    Elixir 
    iex(4)> Enum.each(
          1..3,
          &IO.puts/1
        )
1
2
3

    Internally, a range is represented as a map that contains range boundaries. The memory footprint of a range is very small, regardless of the size. A million-number range is still just a small map.

    Keyword Lists

    A keyword list is a special case of a list, where each element is a two-element tuple, and the first element of each tuple is an atom. The second element can be of any type.

    Elixir 
    iex(1)> days = [{:monday, 1}, {:tuesday, 2}, {:wednesday, 3}]

    Elixir allows a slightly more elegant syntax for defining a keyword list:

    Elixir 
    iex(2)> days = [monday: 1, tuesday: 2, wednesday: 3]

    Both constructs yield the same result: a list of pairs.

    Keyword lists are often used for small-size key/value structures, where keys are atoms.

    Fetching

    You can use Keyword.get/2 to fetch the value for a key:

    Elixir 
    iex(3)> Keyword.get(days, :monday)
1

iex(4)> Keyword.get(days, :noday)
nil

    As with maps, you can use the operator [] to fetch a value:

    Elixir 
    iex(5)> days[:tuesday]
2

    Because youre dealing with a list, the complexity of a lookup operation is O(n).

    Usage

    Keyword lists are most often useful for allowing clients to pass an arbitrary number of optional arguments.

    For example, the result of the function IO.inspect, which prints a string representation of a term to the console, can be controlled by providing additional options through a keyword list:

    Elixir 
    iex(6)> IO.inspect([100, 200, 300])               
[100, 200, 300]    
                               
iex(7)> IO.inspect([100, 200, 300], [width: 3])   
[100,                                             
 200,                                             
 300]

    The pattern is so frequent that Elixir allows you to omit the square brackets if the last argument is a keyword list:

    Elixir 
    iex(8)> IO.inspect([100, 200, 300], width: 3, limit: 1)
[100,
 ...]

    Here you are still sending two arguments to IO.inspect/2: a number and a two-element keyword list.

    For optional arguments, you can accept a keyword list as the last argument of your function, and make that argument default to an empty list:

    Elixir 
    def my_fun(arg1, arg2, opts \\ []) do
  ...
end

    Your clients can then pass options via the last argument.

    Keyword Lists vs Maps

    Compared to maps a keyword list

    • Can contain multiple values for the same key.
    • In addition, you can control the ordering of keyword list elements something that isnt possible with maps.

    MapSet

    A MapSet is the implementation of a set a store of unique values, where a value can be of any type.

    Elixir 
    iex(1)> days = MapSet.new([:monday, :tuesday, :wednesday])    
#MapSet<[:monday, :tuesday, :wednesday]>

iex(2)> MapSet.member?(days, :monday)                         
true    
                                                      
iex(3)> MapSet.member?(days, :noday)                          
false              
                                           
iex(4)> days = MapSet.put(days, :thursday)                    
#MapSet<[:monday, :thursday, :tuesday, :wednesday]>

    You can manipulate the set using the function from the MapSet module

    Enumerability

    A MapSet is also an enumerable, so you can pass it to functions from the Enum module:

    Elixir 
    iex(5)> Enum.each(days, &IO.puts/1)
monday
thursday
tuesday
wednesday

    MapSet (as with Maps) doesnt preserve the ordering of the items!

    For more functions available in the MapSet module, see:

    Times and Dates

    Elixir has a couple of modules for working with date and time types: Date, Time, DateTime, and NaiveDateTime.

    A date can be created with the ~D sigil. The following example creates a date that represents January 31, 2018

    Elixir 
    iex(1)> date = ~D[2018-01-31]
~D[2018-01-31]

    Once youve created the date, you can retrieve its individual fields:

    Elixir 
    iex(2)> date.year
2018

iex(3)> date.month
1

    Similarly, you can represent a time with the ~T sigil, by providing hours, minutes, seconds, and microseconds:

    Elixir 
    iex(1)> time = ~T[11:59:12.00007]

iex(2)> time.hour
11

iex(3)> time.minute
59

    In addition to these two types, you can also work with datetimes using the NaiveDateTime and DateTime modules. The naive version can be created with the ~N sigil

    Elixir 
    iex(1)> naive_datetime = ~N[2018-01-31 11:59:12.000007]

iex(2)> naive_datetime.year
2018

iex(3)> naive_datetime.hour
11

    The DateTime module can be used to work with datetimes in some timezone. Unlike with other types, no sigil is available. Instead, you can create a datetime by using DateTime functions:

    Elixir 
    iex(4)> datetime = DateTime.from_naive!(naive_datetime, "Etc/UTC")

iex(5)> datetime.year
2018

iex(6)> datetime.hour
11

iex(7)> datetime.time_zone
"Etc/UTC"
